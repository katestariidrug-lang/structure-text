# RUNBOOK: перенос проекта в новый чат

Этот файл определяет **обязательный порядок действий** при работе с проектом
Structure-Text Orchestrator в новом чате ChatGPT.

## Обязательные файлы для загрузки

При переносе проекта в новый чат ОБЯЗАТЕЛЬНО загрузить:

1. `state.json`  
   Путь: `jobs/{job_id}/state/state.json`  
   Назначение: единственный источник истины о текущем состоянии пайплайна.

2. Артефакт PASS 2 — `structure_spec`  
   Путь: `jobs/{job_id}/output/pass_2_execute__out.json`

3. Результат PASS 3 — VERIFY issues  
   Источник:
   - `passes.verify.issues` внутри `state.json`
   - или отдельный артефакт PASS 3 (если используется)

4. `README.md`  
   Используется как источник истины о правилах проекта.

Отсутствие любого из этих файлов делает работу пайплайна недетерминированной.

---

## Правило старта

Если PASS 3 (VERIFY) выполнен для текущего job:

- ЗАПРЕЩЕНО начинать с PASS 1 (DECIDE) в этом же job
- ЗАПРЕЩЕНО начинать с PASS 2 (EXECUTE) в этом же job

Допустимые действия:
- PASS 4 (PATCH)
- анализ и принятие зафиксированных рисков

Если требуется перезапустить PASS 1–3 «с нуля» (например, поменялась каноника формата или входные данные):
- создаётся **новый job** (job_0002, job_0003, …)
- старый job сохраняется как исторический артефакт и не “переписывается задним числом”

Любая пересборка архитектуры внутри уже проверифицированного job считается нарушением state-driven пайплайна.
